Using the 'find' command

On this page, I'll be talking about how to use the find command.

This is used to quickly and efficiantly scan through our file system to find files and directories that meet a certain criteria. We can even perform actions on those results.

Like stated in the contents page, all this information can also be quickly found by typing in $ man find on the command line.

Find (current directory)

I can use this to find all files and sub directories starting within current working directory by using this command

$ find .

Find (specific directory)

I can use find to find files and folders within specific directories by specifying the directory. i.e:

$ find directoryOne

Find (filtered)

I can filter the type of content I search for by using the '-type' command along with the specification 'd'. i.e:

$ find . -type d

I can do the same with files by just switching out the 'd' with an 'f'. i.e:

$ find . -type f

Find files (specific name)

Let's say I know that somewhere within the directory I am in, I am aware of a file that exists by a certain name, but don't know where it would be. I could use:

$ find . -type f -name "test_1.txt"

If I only know part of the name, but not the whole file name, I could use the broad word followed by an '*'. The 'name' parameter here is case sensitive, btw:

$ find . -type -f -name "test*"

If you want the case insensitive version, you would search with the 'iname' parameter instead. i.e:

$ find . -type -f -iname "test*"

Using the '*'

The star is a wildcard statement. You can even use it when searching for all files with a specific file type. i.e:

$ find . -type f -name "*.py"

This would find all the python files in the scope of the current directory.

Finding files through metadata

I can find files that were modified in the last 10 minutes by using:

$ file . -type f -mmin -10

'-10' will find files modified less than 10 minutes ago

'+10' will find files modified more than 10 minutes ago

I can also combine these options so i can find files modified between a certain time by doubling the '-mmin' parameter. i.e:

$ find . -type f -mmin +10 -mmin -40

This would find files modified between 40 minutes and over 10 minutes ago.

Sometimes working with minutes is not ideal. For example, imagine calculating the minutes between now and 20 days ago.... That's not exactly a cause for celebration.

for starters, we could filter using '-mtime'. i.e:

$ find . -type f -mtime -12

This would find files modified at any point before 12 days ago.

you can also use this parameter the same way you use the 'mmin' parimeter. i.e: 'mtime +20' would search for modifications over 20 days ago. you could also combine the parameters to specify a search.

Other parameters you could used

access date

-atime

accessed minutes

-amin

changed minutes

-cmin

changed days

-ctime

Finding files by file size

You would use the 'size' parameter. i.e, finding files over 5mb:

$ find . -size 5M

This uses similar syntax to the time synax [time uses +/- for after/before].

Upper case M = megabytes

Upper case G = gigabytes

Lower case k = kilobytes

Finding empty files

Probably one of the most useful features of the find command is finding empty files. to use this:

$ find . -empty

Search based on permissions

If you need to search for files based on permissions, you can use:

'777' allows anyone to view these files

$ find . -perm 777

Changing user and group permissions

Follow these 3 steps

Find the directory you want to change

$ find dir_name

Then use the execute command to change the owner

The 'chown' command is change owner and you're stating that the files owned by this person will be changed

$ find dir_name -exec chown user:group

You need to add a placeholder for this to work

the placeholder here would be the curly brackets and the "+" ends the script. you could also use "/;"

$ find dir_name -exec chown user:group {} +

Setting directory permissions to 775 and file permissions to 664

The directory change:

Notice that the difference here is that you include search parameters that either state files only or directories only.

The 'chmod' works like the 'chown' command

$ find dir_name -type d -exec chmod 775 {} +

Changing the file permissions

change the d to an f...

$ find dir_name -type f -exec chmod 664 {} +

Deleting files that end with a certain extention

follow the 3 steps

Find the file types in the destination you want the action to occur

Make sure to do this step to ensure you're not deleting or changing anything important

$ find . -type f -name "*.jpg"

If you don't want to delete files in your hidden directories, set a 'maxdepth' perimeter.

$ find . -type f -name "*.jpg" -maxdepth 1

'maxdepth' and setting it to '1' means means you'll only search down 1 directory which means the current directory

Then run your execution after you have the correct results

here the execution code is 'rm' for remove. you then add the placeholder and the 'end script' command [the '+']

$ find . -type f -name "*.jpg" -maxdepth 1 -exec rm {} +
