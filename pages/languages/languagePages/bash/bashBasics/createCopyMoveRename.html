Copy, Move, Rename and Delete

Here I will be talking about; creating files and directories, moving files and directories, copying files and directories, re-naming files and deleting files and directories all from your terminal. Sounds like a dream, right?

Creating Directories

This is pretty straightforward. So, to create a directory i will use the command 'mkdir' and your new directory name;

$ mkdir directoryName

Creating Files

This is also pretty straightforward. You just use the 'touch' command and the new file name written with the new file's filetype. i.e:

$ touch testFile.txt

Copying Files

To copy files, you use the 'cp' command followed by the file you want to copy. This will duplicate the file within the current directory

$ cp test_file.txt

Copying Directories

This is a little bit different to copying files.

Method 1

First, create the file location you want to copy to using 'mkdir' [in the current directory]

Then use the command 'cp', then the command '-R', and then the file you're copying, and then the new directory you're copying the files to. i.e

$ cp -R old/ new/

Renaming a file

Renaming files is a little wierd. You use the 'mv' command and move the file to the same directory with a different name. You do this by stating which file you want to move and then the file name you want it to have i.e:

$ mv test.txt newFile.txt

Renaming a Directory

Same as renaming a file. i.e

mv oldDir/ newDir

Moving a file without renaming the file

We do this by stating the new location and no file name

To move a file into a location within the file's directory, you use the command:

$ mv test.txt subDirectory1/

To move the file to a parent directory, you would use:

$ mv test.txt ../

Moving a Directory without renaming it

The process is the same as moving files.

Moving the directory into a subdirectory:

ps, if the file you're moving isn't empty, you need to add the file name to the sub directory declaration. i.e subDir/movedDir

$ mv newDir/ subDirectory1/

Moving the directory into a parent Directory:

$ mv newDir/ ../

Moving a file and renaming the file

To move the file into a sub directory within the current directory, you would use the command:

$ test.txt subDirectory1/newName.txt

to move the file into the parent directory with a new name, you would use:

$ mv newName.txt ../test.txt

Moving a Directory and renaming it

Same process as moving the files and renaming it

To move and rename the directory to a sub directory:

$ mv oldDir subDirectory/newDirName

To move the directory to a parent directory:

$ mv oldDir ../newDirName

Deleting Files

To delete files, you use the 'rm' command and name the file you want to remove

Warning: Deleting files from the terminal is permanent. There is no warnings given when deleting files and also, there is no recovering the files from your trashcan folder

$ rm test.txt

Deleting Directories

Similar process to copying a directory. When deleting a directory, you need to add the '-R' to the 'rm' command. i.e:

$ rm -R deleteDirectory

You might run into some files that refuse to delete. For this use the '-rf' command. But as always, be careful as these files will never come back....unless you have a backup somewhere

$ rm -rf forceDeleteDirectory
