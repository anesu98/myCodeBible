Navigating Your File System

Here, I'm going to be talking about the navigation commands that you need to know to move around comfortably in the terminal

The series I'll be using for these pages will be in the techical documentaion section

Relative Paths

These are used together with some of the other movement commands to let the terminal know which directory you are refering to

One fullstop would refer to the current directory

.

2 fullstops would refer to the parant directory (previous directory).

..

Print Working Directory

This command will print where we currently are on the file system in your terminal.

$ pwd

Listing Items

This command will list all of the files currently located in the directory that you are in on your terminal.

$ ls

The command 'ls' alone doesn't show the hidden files in your directory. To see these you need to use;

$ ls -a

This will list out more information about the files in your directory by doing so in long form. It will show you data information, permissions, user/group owners and sizes information.

$ ls -l

Combining commands 'la' and 'ls' will show you the long form information of all files AND hidden files.

$ ls -la

Change Directory

This command will do one of 2 things.

Using 'cd' alone will take you to the main directory.

$ cd

You can also travel to your home directory by using a '~' after you use the cd command

$ cd ~

Using 'cd' and a file location within the current directory you are in will take you to that file.

$ cd thisFileRightHere

To navigate backwards to our parent directory, you would use 'cd' with 2 fullstops (the parent directory relate path)

$ cd ..

You can travel through multiple directories if you know the name of the directories by splitting them with a '/'. In the example below, this will take you forward by 2 directories.

$ cd randomFolder/folderInsideRandomFolder

If you want to go back more than one folder at a time, you can use a similar principle where you chain '..'s with a '/' to go back folders. i.e: will take you back 2 directories.

$ cd ../../

Using the '~' command with a '/' command, you can direct yourself to a completely different directory starting from the path in your home directory. i.e lets say I'm in working on one project and I quickly need to switch to a different one. If I know my 'repos' folder is on my home directory, I can just switch to my repos folder by using this command: PS... don't include the space between the '~' and the '/'

$ cd ~ /repos/
