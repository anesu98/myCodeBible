The Grep Command

This stands for 'Global Regular Expression Print'

This command allows us to search for some text within files in a system and is commonly used if you know how to use it.

For this example, the imaginary example file will include a bunch of random names and dummy data and will be named 'names.txt'.

Searching for some text within a normal file

An easy example to show what you can do with the GREP command. Let's say we're searching for the name 'Jane Williams'. Here we use the command, what we want so search in quotes and the location we want to search;

$ grep "Jane Williams" names.txt

If we want to search and return a the match and not any results which include the searched phrase, we add the '-w' before our search term. This means search only these whole words;

$ grep -w "Jane Williams" names.txt

GREP is case sensitive, so to make it not case sensitive, you can add an 'i' to the '-w' parameter.

$ grep -wi "Jane Williams" names.txt

Sometimes we want some extra information, for example, the line in which the name appears. to do this, we can add an 'n' to the last funtion for example.

$ grep -win "Jane Williams" names.txt

Just so you know, each letter added is it's own parameter. for example -i and -n. they just join together

Searching for text/code with context

Let's say you're searching a bit of computer code and want to see the context (the surrounding code) of what you just searched. you can use:

To search for the context before the searched term is mentioned, use the parameter '-B' and the number of lines you want to see e.g to see 4 lines before;

$ grep -win -B 4 "Jane Williams" names.txt

To see the context after the searched term, use the '-A' parameter with the number of lines you want to see. e.g;

$ grep -win -A 4 "Jane Williams" names.txt

To see the context before and after at the same time, use the '-C' parameter;

$ grep -win -C 4 "Jane Williams" names.txt

Search for text in every file on a system, not just one

I know that just changing the file name to './' might make sense, but that would get you an error. To do this, you'd need to add an asterisk wildcard, for example;

$ grep -win "Jane Williams" ./*

This will search evrything including any sub directories in the current directory and that in turn would return an error. Or the searched term might bring up too many files at once. To narrow your search, you can add a file type after the '*', e.g;

$ grep -win "Jane Williams" ./*.txt

Maybe we want to search those sub directories to because you genuinely don't know what you're looking for. you can use a recursive search that uses a '-winr' parameter. This starts the search starting from the mentioned directory;

$ grep -winr "Jane Williams" ./

notice how we no longer need the *

Search every file, but only return the file name

Let's just say you don't need to know anything else. Just the file name, you can remove the -n because you need the line number anymore and rthen add a -l to the end. e.g:

$ grep -wirl "Jane Williams" ./

You can see how many times the searched term appears in a file by changing the -l to a -c. e.g:

$ grep -wirc "John Williams" ./

Search our command line history using GREP

You can use type in history in your terminal to reveal what you've been typing in. however, let's say you and to only search your git commits on the device. you can use grep by typing in:

$ history | grep "git commit"

If I only want to see commits including certain words in the messages, I can add another grep search i.e:

$ history | grep "git commit" | grep "added"

GREP Versions

The difference btween using grep on man and linux is that linux uses gnu grep and mac uses bde grep. That just means that using pearl expressions on mac won't work without a lot of work around. If this doesn't bother you (which at the time of writing it doesn't) don't worry. If it does, then use homebrew to install gnu grep on mac

if you have gnu grep, you can use pearl accepted expressions by using a -p parameter. i.e:

grep -p "\d(3)-\d(3)-\d(4)"

This should return a phone number written in any files that have written a phone number in this format now.
